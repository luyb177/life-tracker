# Life Tracker 需求文档

## 1. 项目概述

Life Tracker 是一个个人生活记录系统，核心目标是帮助用户持续记录日常行为与支出，并在此基础上通过 AI 生成周期性总结与建议。

### 1.1 产品目标
1. **高频记录**：支持用户随时记录当天做了什么。
2. **精细记账**：支持用户记录每日多笔支出，并按类别管理。
3. **周期总结**：基于记录数据，自动生成日 / 周 / 月 / 年 / 人生维度的总结与建议。

## 2. 用户定位

### 2.1 目标用户
- 有自我管理需求的个人用户
- 希望同时管理生活记录与消费记录的用户
- 希望通过 AI 获得复盘建议的用户

### 2.2 使用场景
- 随手记录刚完成的事情
- 吃饭、购物、杂项消费后立刻记账
- 晚上查看今日总结
- 周末/月末查看阶段性复盘与趋势变化

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 生活记录
支持用户记录每天做了什么。

**需求点：**
- 支持一天内多次记录
- 每条记录至少包含：
  - 记录时间
  - 内容描述
  - 可选标签（如工作、学习、运动、娱乐、睡眠）
- 支持查看当天所有记录
- 支持按日期查看历史记录
- 支持编辑和删除记录

#### 3.1.2 支出记录
支持用户记录每天花了多少钱。

**需求点：**
- 支持一天内多笔支出
- 默认分类包括：
  - 早饭
  - 午饭
  - 晚饭
  - 杂项
- 支持用户预设常用支出项（如吹头发、咖啡、打车等）
- 每笔支出至少包含：
  - 金额
  - 分类
  - 时间
  - 备注（可选）
- 支持查看当天总支出
- 支持按分类汇总支出
- 支持编辑和删除支出记录

### 3.2 增强功能

#### 3.2.1 AI 周期总结
系统根据用户记录自动生成总结与建议。

**总结周期：**
- 今日总结
- 本周总结
- 本月总结
- 本年总结
- 人生总结

**总结内容建议包含：**
- 吃了什么
- 花了多少钱
- 睡了多久
- 运动了多久
- 做了哪些高频事项
- 消费结构分析
- 生活习惯趋势分析
- AI 改进建议

**输出要求：**
- 总结内容可读性强
- 有事实归纳，也有建议输出
- 支持手动触发和定时生成

#### 3.2.2 统计分析
为 AI 总结和用户复盘提供结构化数据支持。

**需求点：**
- 日 / 周 / 月支出统计
- 分类支出占比统计
- 活动标签统计
- 睡眠 / 运动等长期趋势统计（依赖后续结构化记录）

## 4. 非功能需求

### 4.1 易用性
- 记录入口要足够轻量
- 常用分类和预设项尽量减少输入成本
- 移动端优先，适合随手记录

### 4.2 性能
- 单次记录提交响应时间 < 1 秒
- 常规列表查询响应时间 < 2 秒

### 4.3 可扩展性
- 支持未来新增更多记录类型，如睡眠、运动、体重、心情
- 支持后续接入更多 AI 分析能力

### 4.4 数据安全
- 用户数据需具备基础鉴权保护
- 涉及 AI 调用时应避免泄露敏感信息
- 需要具备基础备份能力

## 5. 技术选型

当前 README 已明确的方向：
- **后端**：Go、go-zero、gorm
- **前端**：Vue 3

在此基础上建议补全如下：

| 层级    | 技术选型                     | 说明                   |
|-------|--------------------------|----------------------|
| 后端语言  | Go                       | 性能稳定，适合构建 API 服务     |
| 后端框架  | go-zero                  | 适合快速搭建服务、路由、鉴权与微服务扩展 |
| ORM   | GORM                     | 适合快速建模与常规 CRUD       |
| 数据库   | MySQL                    | 易开发                  |
| 缓存/任务 | Redis                    | 用于定时任务、热点缓存、任务状态管理   |
| 定时调度  | go-zero + cron / 独立调度器   | 用于 AI 总结定时生成         |
| 前端框架  | Vue 3                    | 适合构建中后台/移动 Web 界面    |
| UI 框架 | Naive UI                 | 适合快速搭建录入与统计页面        |
| 状态管理  | Pinia                    | Vue 3 主流方案           |
| 网络请求  | Axios                    | 常规 API 请求处理          |
| 图表    | ECharts                  | 适合支出趋势与分类占比展示        |
| AI 接入 | OpenAI 兼容接口 / 自定义 LLM 服务 | 用于生成周期总结与建议          |
| 部署    | Docker + caddy           | 便于后续部署与维护            |

### 5.1 推荐技术方案结论
1. **数据库推荐 Mysql**：更利于后续统计分析与时间维度查询。
2. **缓存推荐 Redis**：支撑定时任务、AI 总结队列、热点数据。
3. **前端优先 Web + 移动端适配**：先做响应式页面，后续再考虑小程序或原生 App。
4. **AI 模块独立设计**：避免把总结逻辑耦合进核心记录流程。

## 7. 实现优先级

### P0（必须优先实现）
这是最小可用版本（MVP）。

#### 7.1 基础记录能力
- 用户创建生活记录
- 用户查看某天生活记录
- 用户新增支出记录
- 用户查看某天支出记录
- 默认支出分类支持
- 支出汇总（日总额）

#### 7.2 基础数据管理
- 记录编辑 / 删除
- 日期维度查询
- 简单列表页和录入页

**目标：**
先把“能记、能查、能改”做好，确保产品可实际使用。

### P1（第二阶段）
在 MVP 可用后尽快补齐效率与体验。

#### 7.3 自定义能力
- 用户自定义常用支出分类
- 常用记录模板 / 快捷输入
- 标签管理

#### 7.4 基础统计
- 按天 / 周 / 月统计支出
- 分类支出占比
- 高频活动标签统计

**目标：**
让产品从“能记录”升级到“更方便复盘”。

### P2（第三阶段）
开始引入 AI 能力。

#### 7.5 AI 总结能力
- 手动触发今日总结
- 手动触发本周 / 本月总结
- AI 总结结果落库
- 总结页面展示

#### 7.6 定时任务
- 每日定时总结
- 每周定时总结
- 每月定时总结

**目标：**
验证 AI 总结是否真正有价值。

### P3（第四阶段）
做深度分析与长期能力建设。

#### 7.7 长期趋势分析
- 年度报告
- 人生维度长期总结
- 消费趋势分析
- 生活习惯变化识别

#### 7.8 扩展记录类型
- 睡眠记录
- 运动记录
- 体重记录
- 心情记录

**目标：**
从“记录工具”发展为“个人生活分析系统”。

## 8. 里程碑建议

| 阶段 | 目标       | 核心交付                |
|----|----------|---------------------|
| M1 | 完成 MVP   | 生活记录 + 支出记录 + 查询/编辑 |
| M2 | 完成统计能力   | 自定义分类 + 统计面板        |
| M3 | 完成 AI 初版 | 手动总结 + 定时总结         |
| M4 | 完成长期分析   | 年度趋势 + 多维记录扩展       |

## 9. 验收标准

### 9.1 MVP 验收
- 用户能在 1 分钟内完成当天生活记录和支出记录
- 能查看任意一天的记录与支出汇总
- 能编辑和删除已有记录

### 9.2 AI 阶段验收
- 系统可基于指定周期数据生成总结
- 总结内容包含事实归纳和建议
- 定时任务可稳定执行并保存结果
